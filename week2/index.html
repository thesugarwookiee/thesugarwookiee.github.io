Novice to Ninja, Chapter 2

  • NON-VARIABLES: abstract, await, boolean, break, byte, case, catch, char, class, const, continue, debugger, default, delete, do, double, else, enum, export, extends, false, final, finally, float, for, function, goto, if, implements, import, in instanceof, int, interface, let, long, native, new, null, package, private, protected, public, return, short, static, super, switch, ssynchronized, this, throw, throws, transient, true, try, typeof, var, volatile, void, while, with, yield
  • Also avoid: undefined, NaN, Infinity
  • Data Types: string, symbol, boolean, undefiend, null, number. Anything not these, such as arrays, functions and object literals are called objects.
  • Const cannot have a reassigned value later in the code, let can. Const can be mutated, however. Const is better used to avoid bugs.
  • Var is outdated in javascript and I didn't know that.
  • If you define a variable, even within a block, without let or const, it will be a global variable. Consider scope, always!
  • Is there a reason to not use Template Literals always? They seem very convenient.
  • Boolean values, always falsy: blank strings or templates, 0, -0, NaN, false, null, and undefined.

Chapter 3

  • Arrays go in square brackets! Literals are easier and better than making arrays with functions.
  • Arrays are special built-in objects rather than primitive data.
  • You can assign to the array directly with an index number and/or replace index values.
  • .pop removes end of array, .shift removes front, .push adds to end of array, .unshift adds to front, .join makes a string of the contents of the array, .slice chops out a piece of the array (non-destructive), .splice takes out a chunk of array and replaces it with a new value (or not)
  • Sets are useful for elimating duplicates in arrays.
  • Maps are basically arrays with keys and values? Unclear, need to look into this more.
