  • An object literal is an object that is created directly in the language by wrapping all its properties and methods in curly braces.

Chapter 6

  • DOM treats the HTML as the root tag and everything else as children and children of children and so forth.
  • DOM can be used with any programming language. Javascript accesses it through a special object called document.
  • Node lists are array like objects, but not actually arrays. But you can turn one into an actual array with Array.from()
  • For DOM and javascript reasons, it's very important to only ever have one instance of an ID on any given page.
  • In Javascript you can 'get' elements by their ID, tag, and class. Query selectors are also useful for selecting more specific elements without necessarily putting IDs into your HTML.
  • childNodes returns all nodes, including text nodes children returns only elements of that node and will ignore text nodes.
  • you might expect the first child node of the ul element to be a li element, and the last child to also be a li element, but they are both in fact text nodes, generated by the whitespace characters in between the ul and li tags
  • wonderWoman.setAttribute('class', 'villain'); is how you set attributes, with two arguments.
  • wonderWoman.className = 'hero' can also be used to set a new class, but will overwrite all existing classes unlike setAttribute. classList.add can also be used to add without altering the others, as can classList.remove. classList.toggle will remove if present and add if not.
  • Don't forget .display = "none" when you want to hide something until or after some event.

Chapter 7

  • Speaking of events, here are events.
  • document.body.addEventListener("click", doSomething); here's the basic idea of one for clicking anywhere on the page.
  • inline event listeners do look but aren't recommended for a number of reasons, including the fact that it's very limiting.
  • There are many types of events, but for now it's easier to focus on interactions from input devices like the keyboard and mouse.
  • mouseup, mousedown, click all are mouse events. also dblclick for special occasions. mouseover is what it sounds like it is, as it mouseout and mousemove.
  • keydown, keypress, and keyup are all keyboard events. keydown is the actual pressing, keypress is actually typing something on the screen.
  • touchstart and touchend are touch controls but click may be a better option in many cases
  • if implimented correctly, removeEventListener can be used to make something only executable one time.
  • preventDefault() exists and can stop default behavior (such as following a link) from happening upon an event, but likely will have little application in my own projects.
